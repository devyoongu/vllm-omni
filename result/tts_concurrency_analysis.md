## 2. GPU 부하 분석

### 2.1 Active 구간 요약

| 구간 | 시각 | 지속 시간 | max util | max power | avg power |
|:---:|------|:--------:|--------:|----------:|----------:|

| 3개 요청 | 16:03:40.4 ~ 16:03:46.3 | **5.9 s** | 98 % | 319 W | 248 W |
| 5개 요청 | 16:03:57.3 ~ 16:04:05.2 | **7.9 s** | 100 % | 320 W | 258 W |

> RTX 3090 TDP: 350 W / VRAM 사용량: 16,202 MiB (전 구간 고정)

### 2.2 GPU Utilization 패턴

**1개**: util 최대 89%, 단일 burst 후 종료. PCIe gen1(저전력) → gen3 전환 발생.

**3개**: util 98~99% 지속. 배치가 균일하게 처리되어 util이 단일 블록으로 유지됨.

**5개**: util 100% → 42% → 89% → 42% 반복 패턴.
서로 다른 텍스트 길이로 인해 Thinker(AR) / Talker(Code2Wav) 단계 전환 타이밍이 요청마다 달라지고, GPU가 배치 경계에서 idle 구간을 가지며 비균일한 처리가 발생.

### 2.3 전력 효율

```
1개: avg 196 W  → GPU 활용률이 낮아 전력 대비 효율 낮음
3개: avg 248 W  → 배치 처리 효율 양호
5개: avg 258 W  → 전력은 비슷하지만 처리 시간이 길어짐 (util 불균일)
```

## 3. Real-Time Factor (RTF) 분석

> RTF = 오디오 길이 / 총 소요 시간. 1.0 미만이면 실시간 처리 불가.

| 동시 요청 | 가장 긴 오디오 | 해당 요청 총 소요 | RTF |
|:--------:|:------------:|:--------------:|:---:|
| 1개 | 6.6s | 2,760ms | **2.39x** |
| 3개 | 9.6s | 5,512ms | **1.74x** |
| 5개 | 10.4s | 8,106ms | **1.28x** ← 실시간 마진 감소 |

5개 동시에서 RTF 1.28x — 여유가 작아 콜봇처럼 응답 지연에 민감한 환경에서 위험 구간에 접근.

## 4. 결론 및 권장사항

### 4.1 최적 동시 요청 수

| 구분 | 권장 값 | 근거 |
|------|:------:|------|
| RTX 3090 단일 GPU 안정 운영 | **3개** | util 98%로 균일, TTFA +32% 수준 허용 가능 |
| 최대 허용 (음질·지연 감수) | **5개** | 성공률 100%이나 TTFA +68%, RTF 마진 축소 |
| 위험 구간 | **7개 이상** | 서버 hang 현상 확인됨 (스케줄러 포화 추정) |

### 4.2 TTFA 증가 특성

- 1개 → 3개: TTFA +32% — 콜봇 UX 허용 범위 (392ms → 516ms, +124ms)
- 1개 → 5개: TTFA +68% — 체감 가능 수준 (392ms → 657ms, +265ms)
- TTFA는 총 소요 시간보다 증가 폭이 작음 (배치 첫 청크는 상대적으로 빨리 생성)

### 4.3 메모리

VRAM 16,202 MiB가 요청 수 전 구간에서 고정 — KV cache가 사전 할당되어 메모리 병목 없음.
추가 확장 시 VRAM 제약보다 GPU 연산 처리량(utilization)이 먼저 병목이 될 것.
