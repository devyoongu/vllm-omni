## 3. Real-Time Factor (RTF) 분석

> RTF = 오디오 길이 / 총 소요 시간. 1.0 미만이면 실시간 처리 불가.

| 단계 | 최장 오디오 | 해당 요청 총 소요 | RTF | 여유 |
|:---:|----------:|:--------------:|:---:|:---:|
| 1개 | 6.5s | 2,673ms | **2.43x** | 충분 |
| 2개 | 10.2s | 4,620ms | **2.21x** | 충분 |
| 3개 | 8.9s | 5,299ms | **1.68x** | 양호 |
| 4개 | 10.3s | 6,820ms | **1.51x** | 주의 |
| 5개 | 10.7s | 8,070ms | **1.33x** | 경계 |

RTF가 1.0에 가까워질수록 스트리밍 중 버퍼 언더런(끊김) 위험 증가.
5개 동시에서 1.33x — 텍스트가 더 길거나 서버 부하 가중 시 1.0 이하로 떨어질 수 있음.

### 4.2 운영 권장 기준 (RTX 3090 단일 GPU)

| 구분 | 동시 요청 수 | 근거 |
|------|:-----------:|------|
| 최적 (안정+효율) | **2~3개** | avg util 최고(80%), TTFA +32% 이하, RTF 1.68x 이상 |
| 허용 최대 | **4개** | 성공률 100%, RTF 1.51x — TTFA +59% 허용 가능 수준 |
| 위험 경계 | **5개** | RTF 1.33x, TTFA 2배 이상 — 텍스트 길이에 따라 불안정 |
| 금지 | **7개 이상** | 이전 테스트에서 서버 hang 확인 |

### 4.3 추가 관찰

- **VRAM 고정**: 16,200~16,202 MiB — 요청 수와 무관하게 고정. 메모리는 병목이 아님
- **PCIe 상태**: 전 구간 gen3 유지 (이전 테스트 대비 gen1 저전력 전환 없음 — 서버가 워밍업된 상태로 측정됨)
- **avg util 역설**: 4~5개에서 avg util(62%)이 2개(80%)보다 낮음 — GPU가 더 오래 켜져 있지만 효율적으로 쓰이지 않음. 단계 전환 대기 시간이 전체 처리 시간을 늘리는 구조
